1.1 为什么要用this

显示传递上下文对象会使得代码变得混乱
通过this可以让API更简洁并且易于复用

1.2 误解

1.2.1 指向自身

认为this指向函数自身

1.2.2 它的作用域

认为this指向函数作用域
this在任何情况下都不指向函数的词法作用域
this与词法作用域的查找混合使用容易被迷惑

1.3 this是什么

this实在运行时进行绑定的，它的上下文取决于函数调用时的条件。this的绑定和函数声明的位置没有任何关系，只取决于函数的调用方式

当函数被调用，会创建一个执行上下文（活动记录）。包含函数在哪里被调用（调用栈），函数的调用方法，传入的参数信息，
this被用来记录在哪里被调用

2.2 绑定规则

2.2.1 默认绑定

独立函数调用绑定到全局对象
严格模式：全局对象无法使用默认绑定，this会绑定到undefined
函数体是否处于严格模式

2.2.2 隐式绑定

作为对象属性调用
当函数引用有上下文对象时，隐式绑定将this绑定到上下文对象

隐式丢失

2.2.3 显示绑定

通过call(),apply()显示进行绑定

显示绑定无法解决丢失绑定问题

  1. 硬绑定

硬绑定是常用的模式，所以ES5提供bind方法

硬绑定的两种应用场景

  1. 创建包裹函数，传入所有参数并返回接受到的所有值

  2. 创建一个i可以重复使用的辅助函数

2.2.4 new绑定

使用new调用构造函数

  1. 创建一个新的对象
  2. 新的对象会被执行[[原型]]连接
  3. 新对象会绑定到函数的调用的this上
  4. 如果函数没有返回其他对象，new表达式的函数调用自动返回新的对象
    构造一个新的对象并绑定到构造函数中调用中的this上

2.3 优先级

在new 中使用硬绑定，可以预先设置函数的一些参数，在new进行初始化时就可以只传入其余参数

new,显示绑定,隐式绑定,默认绑定

2.4 绑定例外

2.4.1 被忽略的this

将null或者Undefined作为this绑定对象传入call,apply,bind在调用时会被忽略，实际上是默认绑定规则

传入Null的场景

  1. 数组展开成参数
  2. bind进行柯里化（预设参数）

但是在一些看库中可能产生难以调试的bug

更安全的做法：传入不影响程序的特殊对象，ø空集合符号
Object.create(null),不会创建Object.prototype,

2.4.2 间接引用

2.4.3 软绑定

2.5 词法

箭头函数不使用this的四种标准规则，而是根据外层作用域决定this
箭头函数的this绑定无法修改，包括new

代码风格：

  1. 只是用词法作用域，抛弃this风格
  2. 完全this风格，必要时是用bind，避免self = this 和箭头函数

3.3.4 复制对象

需要保证JSON是安全的

浅复制
Object.assign()
遍历源对象的所有可枚举的自有键，复制到目标对象（使用=操作符赋值，所以源对象属性的特征不会被复制到目标）

3.3.5 属性描述符

获取属性Object.getOwnPropertyDescriptor(obj,key)返回属性描述对象

属性的两种状态

  • 属性描述符
    • 访问描述符

属性描述符

  • value
    • writable 可写
    • enumerable 可枚举
    • configurable 可配置

访问描述符

  • getter
    • setter 可写
    • enumerable 可枚举
    • configurable 可配置

3.3.6 不变性

  1. 对象常量
    writable:false,configurable:false 创建常量属性
  2. 禁止扩展
    添加新属性会报错
    Object.preventExtensions()
  3. 密封
    Object.seal()密封对象
    实际上是对现有对象调用Object.preventExtensions()
    并将所有现有属性标记为configurable:false
  4. 冻结
    Object.freeze()创建冻结对象
    事实上是在现有对象上调用Object.seal()并将所有数据属性标记为writable:false

3.3.7 [[Get]]

语言规范中：[[Get]]操作像是函数调用，现在对象中查找是否有名称相同的属性，找到就返回，没有找到按照[[Get]]算法，在原型链中查找

3.3.8 [[Put]]

[[Put]]并不等于赋值操作
具体行为取决于多种因素

  1. 属性是否是访问描述符，是并且存在setter调用setter
  2. 属性的数据描述符中writable是否是false,是则失败，严格报typeError
  3. 不是，将值设置为属性值

不存在这个属性，将会更复杂。

3.3.9 Getter和Setter

对象默认的[[Set]],[[Put]]操作分别可控制属性值的设置与获取

用过getter,setter改写默认操作，只能应用在单个属性上
给属性设置getter``setter，当前属性被定义为访问描述符
访问描述符js会忽略value,和writable特性，替换为 get,set特性

改写默认取值赋值，根据setter,getter函数返回决定实际值

3.3.10 存在性

判断对象中是否存在这个属性
in操作符，会检查对象以及其原型链,不关心是否可枚举
obj.hasOwnProperty()只会检查自身

Note:通过Object.create(null)创建的对象没有hasOwnProperty方法

  1. 枚举

propertyIsEnumerable(..) 会检查给定的属性名是否直接存在于对象中(而不是在原型链 上)并且满足 enumerable:true。

Object.keys(..) 会返回一个数组，包含所有可枚举属性，Object.getOwnPropertyNames(..) 会返回一个数组，包含所有属性，无论它们是否可枚举。

第四章 混合对象“类”

面向类的设计模式

  • 实例化instantiation
  • 集成inheritance
  • 多态polymorphism 父类的通用行为可以被子类重写

4.1 类理论

描述了一种代码的组织结构
强调的是：数据和操作数据的行为本质上是相互关联的
类理论建议父类和子类使用相同的方法名来表示特定的行为，从而让子类重写父类。在 JavaScript代码中这样做会降低代码的可读性和健壮性。

4.1.1 “类”设计模式

面向对象设计模式: 迭代器模式、观察者模式、工厂模式、单例模式

4.1.2 js中的“类”

js实际上没有类，有近似类的语法

4.2 类的机制

4.2.1 建造

“类”“实例”的概念来源于房屋建造

4.2.2 构造函数

类实例是有特殊的类方法构造的
这个方法通常和类名相同，被称为构造函数

4.3 类的继承

父类与子类并不是实例，子类继承，子类包含父类行为的原始副本

4.3.1 多态

继承父类方法，重写，或者使用继承来的方法。
这种既是被称为多态或虚拟多态,相对多态

super：’超类’，当前类的父类/祖先类

多态：

  • 相对：任何方法都可以易用继承层次中高层的方法，因此相对的意思是查找上一层（不定义访问的绝对继承层次）
  • 不同层次一个方法可以被多次定义

在子类中可以相对引用它继承的父类，这种相对引用被称为super

子类得到的是父类行为的副本，对继承的方法进行重写，不会影响父类中的方法

继承。实际上就是复制

4.3.2 多重继承

多重继承：所有父类定义都会被复制到子类
出现问题，父类相同方法，引用哪个

js不提供多重继承功能

4.4 混入

JS继承或实例化不会复制
js中只有对象，并不存在可以被实例化的类。对象不会被复制，只会被关联

js模拟类的复制行为，这个方法叫做混入。
显式和隐式

4.4.1 显示混入

在很多库和框架中称为extend(),mixin()

  1. 多态
    js中没有相对多态机制，指明调用对象，必须使用绝对引用
    尽量避免显式伪多态(OtherObj.methodName.call(this, ...))

  2. 混合复制
    问题:函数对象复制的是引用
    混入多个对象就实现了多重继承

  3. 寄生继承

  4. 隐式混入

尽量避免，保证代码的整洁和可维护性

5.1 [[Prototype]]

对象的内置属性[[Prototype]]

5.1.2 属性设置和屏蔽

下层会屏蔽原型链上层的同名属性

复杂情况

  1. 原型链上层存在foo，普通数据访问属性，并且不是只读，会直接在myObject上添加一个新的属性，是屏蔽属性
  2. 如果原型链上层存在foo,标记为只读，将无法修改已有属性或者在myObject上创建属性。严格模式下会抛出错误，否则忽略
  3. 原型链上层存在并且为setter会调用setter

可以通过Object.defineProperty()添加属性
尽量避免使用屏蔽

隐式屏蔽

5.2 ‘类’

Js中没有类，只有对象

5.2.1 ‘类’函数

利用函数的特殊特性，函数默认会拥有一个prototype共有不可枚举属性，指向一个对象

函数的prototype是函数的原型属性，当函数作为构造函数时，实例化出来的对象会将函数的原型属性，当做自己的原型对象[[Prototype]]

名称
[[Prototype]]机制，被称为原型继承
在面向类的世界中，继承意味着复制，但js中并没有进行复制

委托: js在两个对象间创建关联，这样对象通过委托原型对象访问它的属性和函数

差异继承: ？？？

5.2.2 ‘构造函数’

构造函数原型上有constructor指向构造函数，通过这个构造函数实例化的对象，通过原型对象也可以访问这个构造函数

  1. 构造函数还是调用
    事实上，new会劫持所有普通函数使用构造对象的形式调用

5.2.3 技术

constructor 与 实例化的对象没有关联关系，当实例化对象的构造函数的原型被替换，在替换之前实例化的对象的原型对象并不会变

手动给构造函数原型的添加constructor构造器，需要手动添加符合正常行为的不可枚举属性

constructor仅仅是不可枚举的

尽量避免这些引用
